# Helpdesk Project — Roadmap

Helpdesk project built with **Python & Django** (using Django’s default database — SQLite). This projects includes **Git, GitHub, and GitHub Actions** to teach collaborative workflows and simple CI/CD.

1. **Project Introduction & Project Overview** — goals, features, tech stack, deliverables
2. **Install Python, pip, and Create a Virtual Environment**
3. **Intro to Git: Install Git & Create a Local Repo**
4. **Intro to GitHub: Create Account, Repo, and Push First Commit**
5. **Project Skeleton: Start Django Project and App**
6. **Django Settings: Debug, Allowed Hosts, Static & Templates Setup**
7. **Modeling: Ticket & UserProfile Models (first pass)**
8. **Database: Migrations, SQLite, and Admin Registration**
9. **Django Admin: Customize Admin for Tickets & Users**
10. **User Authentication: Signup, Login, Logout (basic views)**
11. **Templates: Base Template, Template Inheritance, Bootstrap Intro**
12. **Ticket Creation: Form, View, and Validation**
13. **Ticket List & Detail Views: Querysets and Pagination**
14. **Ticket Assignment & Status Workflow (open, in-progress, closed)**
15. **User Roles & Permissions: Staff vs. Regular Users**
16. **Messaging/Comments on Tickets: Inline Ticket Thread**
17. **File Uploads: Attachments to Tickets (images/docs)**
18. **Email Notifications: Send Emails on Ticket Events (console backend)**
19. **Search & Filters: Filter Tickets by Status, Priority, Assignee**
20. **Profile Page & Settings: Update User Info and Preferences**
21. **Notifications UI: Unread Counts and Simple Alerts**
22. **Activity Log: Record Ticket Events for Audit**
23. **Pagination & UI Polishing: Improve UX for Lists**
24. **Testing Basics: Write Unit Tests for Models and Views**
25. **Static File Management: Collectstatic, CSS, and JS Organization**
26. **Client-side Enhancements: Simple JS for Forms & Live UX**
27. **Security Basics: CSRF, XSS, and Password Handling**
28. **Refactoring: Use Django Class-Based Views & Reusable Templates**
29. **API Introduction: Build Simple Ticket API Endpoints (DRF minimal)**
30. **API Authentication: Token or Session-based access for API**
31. **Documentation: README, Contribution Guide, and Code Style**
32. **Branching Workflow: Feature Branches, Pull Requests, and Reviews**
33. **GitHub Issues & Project Board: Track Tasks and Bugs**
34. **Continuous Integration Basics: Set Up GitHub Actions Workflow (tests)**
35. **CI: Run Linting (flake8/isort) in GitHub Actions**
36. **CI: Run Django Tests on Pull Requests**
37. **CI: Run Static Analysis / Security Checks (bandit basic)**
38. **Protecting Main Branch: Require CI and Reviews on PRs**
39. **Automated Deploy Simulation: Build/Test Job and Artifacts**
40. **Secrets Management in GitHub: Actions Secrets & Safe Usage**
41. **Automated Releases: Tagging and GitHub Actions for Releases**
42. **Collaborative Code Review: Best Practices & Checklist**
43. **Performance Basics: Query Optimization & Indexing Tips**
44. **Error Handling & Logging: Django Logging and User-Friendly Errors**
45. **Backup & Data Export: Simple DB Dump and Restore Instructions**
46. **Preparing for Production: Settings Checklist (without deploying)**
47. **Project Wrap-up: Final Code Cleanup and Documentation**
48. **Course Recap: What You Built & Next Learning Paths**
49. **Capstone Task: Build One New Feature and Submit PR**
50. **Showcase & Feedback: How to Demo, Collect Feedback, and Iterate**
